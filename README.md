# Crime Reporting System Database

## Overview

This database is designed for a **Crime Reporting System** where citizens can report crimes online and police officers can manage and investigate reported incidents.

The database consists of three main tables:

1. **POLICE_INFO** – Stores police officer information.
2. **REGISTERED_USERS** – Stores registered user accounts.
3. **REPORTED_CRIMES** – Stores crime reports submitted by users.

# Database

```sql
CREATE TABLE POLICE_INFO (
    POLICE_ID           VARCHAR2(20)    PRIMARY KEY,
    NAME                VARCHAR2(100),
    FATHERS_NAME        VARCHAR2(100),
    MOTHERS_NAME        VARCHAR2(100),
    PERMANENT_ADDRESS   VARCHAR2(300),
    SELECTION_YEAR      NUMBER(4),
    POSTING_AREA        VARCHAR2(150),
    POSTING_YEAR        NUMBER(4),
    PICTURE_FRONT       BLOB,
    MERITAL_STATUS      VARCHAR2(50),
    INJURIES            VARCHAR2(500),
    POST_NAME           VARCHAR2(50),
    POSTING_CITY        VARCHAR2(100),
    POLICE_STATION      VARCHAR2(100)
);



CREATE TABLE REGISTERED_USERS (
    ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    FULL_NAME VARCHAR2(100) NOT NULL,
    USER_NAME VARCHAR2(100) NOT NULL,
    EMAIL VARCHAR2(150) NOT NULL,
    DOB DATE,
    MOBILE VARCHAR2(20),
    ROLE VARCHAR2(20),
    POLICE_ID VARCHAR2(20),
    PASSWORD VARCHAR2(300) NOT NULL,
    PROFILE_PICTURE BLOB,
    CONSTRAINT FK_REGISTERED_USERS_POLICE
        FOREIGN KEY (POLICE_ID)
        REFERENCES POLICE_INFO (POLICE_ID)
);



CREATE TABLE REPORTED_CRIMES (
    CRIME_ID           NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ID                 NUMBER(5),
    USER_NAME          VARCHAR2(100) NOT NULL,
    FULL_NAME          VARCHAR2(100) NOT NULL,
    ZILLA              VARCHAR2(100) NOT NULL,
    UPAZILLA           VARCHAR2(100) NOT NULL,
    POLICE_STATION     VARCHAR2(100) NOT NULL,
    AREA               VARCHAR2(100),
    ROAD_NAME          VARCHAR2(100),
    ROAD_NO            VARCHAR2(50),
    DATE_OF_INCIDENT   DATE NOT NULL,
    CATEGORY           VARCHAR2(50),
    DESCRIPTION        CLOB,
    STATUS             VARCHAR2(50),
    MEDIA_FILE         BLOB,
    HIDE_IDENTITY      VARCHAR2(5),
    ACCEPTED           VARCHAR2(50) DEFAULT 'Not Accepted' NOT NULL,
    POLICE_ID          VARCHAR2(20),
    UPGRADED_BY        VARCHAR2(100),
    ACCEPTED_BY        VARCHAR2(100),
    MEDIA_TYPE         VARCHAR2(50),
    CONSTRAINT FK_REPORTED_CRIMES_USER
        FOREIGN KEY (ID)
        REFERENCES REGISTERED_USERS(ID),
    CONSTRAINT FK_REPORTED_CRIMES_POLICE
        FOREIGN KEY (POLICE_ID)
        REFERENCES POLICE_INFO(POLICE_ID)
);
```
